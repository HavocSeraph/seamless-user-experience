Section 1 - How to Use This Guide 
This handbook is the authoritative testing reference for SkillBarter - a peer-to-peer skill exchange 
platform powered by a SkillCoin token economy. It is structured as a phase-by-phase prompt suite 
designed to be pasted directly into an AI coding assistant (Claude Opus 4.5, Claude Sonnet 4.5, or 
Gemini 2.5 Pro). Each prompt is self-contained, precise, and written to surface bugs, security 
vulnerabilities, and financial logic errors on the first pass. 
WORKFLOW: Execute phases in order. Each phase depends on the previous. Do not skip Phase 2 
(LedgerService) - it is the financial foundation. A bug in LedgerService means real token loss.

once

Query 2 - Token Conservation Formula 
The master conservation check. The discrepancy column MUST be 0. Any non-zero value indicates 
tokens were created or destroyed outside of defined rules - a critical financial bug. 
Conservation Law: total_issued = current_balances + locked_escrows + burned_tax. If discrepancy != 0, 
check LedgerService for non-atomic operations (sequential awaits instead of prisma.$transaction()).

Query 4 - Orphaned Escrows 
Detects LOCKED escrows that have no associated session or bounty. These represent tokens that 
are permanently locked with no way to release them - a critical data integrity issue. 
Query 5 - Escrow State Machine Violations

(should never happen), or RELEASED/REFUNDED escrows without resolvedAt (should always be 
set).

Query 6 - Duplicate OAuth Bonuses (Idempotency Check) 
Detects users who received the same OAuth bonus more than once - indicating the idempotency 
check in the OAuth strategy failed. Should return 0 rows. 
Query 7 - Admin Wallet Integrity 
Verifies the admin wallet is receiving all tax revenue correctly. The TAX transaction count should 
match the total number of completed sessions and accepted bounties. 
Query 8 - Daily Token Velocity (Anomaly Detection) 
Tracks token flow by day and transaction type over the last 30 days. Use this to detect anomalies: 
sudden spikes in EARN transactions (bonus exploit), unusual ESCROW_LOCK patterns (booking 
fraud), or TAX amounts inconsistent with session volume.
